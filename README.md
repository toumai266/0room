## 2. 개요 (Objective)
"0room"은 일반 사용자와 보안 입문자를 대상으로 하는, 사용자 친화적인 웹 기반 보안 유틸리티 도구 모음입니다. 복잡한 설치나 전문 지식 없이도, 누구나 일상에서 마주치는 디지털 위협을 쉽게 확인하고 예방할 수 있도록 돕는 것을 목표로 합니다.

---

## 3. 대상 고객 (Target Audience)
**일반 사용자**: 스팸 문자나 이메일로 받은 의심스러운 링크(단축 URL)를 클릭하기 전, 안전성을 확인하고 싶은 사람.

**보안/개발 입문자**: 복잡한 전문 툴(e.g., Burp Suite) 대신, 간단한 보안 기능(e.g., 암호 강도, 해시)을 빠르게 웹에서 테스트하고 싶은 학생 및 초보 개발자.

---

## 4. 주요 기능 (Key Features)
"0room"은 사용자가 가장 빈번하게 필요로 하는 핵심 보안 도구를 고도화하여 제공합니다.

### F-01: 암호 보안 수준 검사기 (Password Strength Checker)
**사용자 스토리**: "사용자로서, 내가 사용하려는 비밀번호를 입력하면, 이 비밀번호가 얼마나 안전한지 즉각적으로 피드백을 받고 싶다."

**구현 기능**:
- **실시간 강도 분석**: `zxcvbn` 라이브러리를 활용하여 단순 길이/문자 조합뿐만 아니라, 사전 공격(Dictionary Attack) 취약성까지 분석
- **한글 피드백 제공**: "Straight rows of keys..." 같은 영문 경고를 "키보드 연속 배열은 추측하기 쉽습니다"와 같이 친절한 한글로 번역하여 제공
- **보안 정책 적용**: 최대 30자 길이 제한을 두어 DoS 공격 방지 및 효율성 확보
- **UI/UX**: 입력 즉시 시각적(색상, 게이지)으로 강도를 표시

---

### F-02: 안심 링크 (Safe-Click URL 확장기)
**사용자 스토리**: "사용자로서, bit.ly 같은 의심스러운 단축 URL을 입력하면, 클릭하기 전에 이 링크의 최종 목적지가 어디인지, 그리고 안전한지 확인하고 싶다."

**구현 기능**:
- **URL 경로 추적**: 단축 URL의 리다이렉트 경로를 끝까지 추적하여 최종 목적지(Final Destination) 표시
- **악성 사이트 탐지 (고도화)**:
  - **Phishing Database 연동**: 오픈소스 피싱 도메인 데이터베이스(약 50만 개)를 실시간으로 대조하여 악성 여부 판단
  - **키워드/패턴 분석**: URL 내 `login`, `bank`, `phishing` 등 의심스러운 키워드 및 IP 주소 직접 사용 여부 탐지
- **안정적인 에러 핸들링**: 접속 불가능한 사이트(DNS 오류, 서버 다운)라도 URL 자체 분석 결과를 제공하며, 명확한 한글 에러 메시지 표시
- **누적형 UI**: 채팅처럼 검사 결과가 누적되어, 여러 링크를 연속으로 검사하고 비교 가능

**보안 강화 요소**:
- **HTTPS 여부 표시**: 최종 URL이 암호화된 연결인지 확인
- **타임아웃 설정**: 악의적인 서버로 인한 무한 대기 방지 (5초 제한)

## 5. 향후 확장 계획 (Roadmap / Post-MVP)
MVP 출시 이후, 다음과 같은 도구들을 순차적으로 추가합니다.

**Phase 2**:
- **해시 생성기**: 텍스트를 입력받아 MD5, SHA-256 등 주요 해시 값을 생성.
- **보안 헤더 스캐너**: 웹사이트 URL을 입력받아 HTTP 보안 헤더(e.g., X-Frame-Options, CSP) 설정 유무를 O/X로 표시.

**Phase 3**:
- **EXIF 뷰어**: 이미지 파일을 업로드하면 GPS 위치, 기기 정보 등 메타데이터를 추출해 표시.
- **(심화) "안심 링크" + VirusTotal API 연동**: F-02에서 찾은 최종 URL을 VirusTotal API로 2차 검사하여, 알려진 악성 사이트인지 여부를 함께 표시.

**Phase 4**:
- 사용자 대시보드: 검사 이력 저장 (로그인 기능 필요)
- REST API 제공: 다른 개발자가 0room 기능을 활용할 수 있도록 API 엔드포인트 공개

## 6. 기술 스택 (Tech Stack)
- **백엔드**: Python 3.x, Flask
- **프론트엔드**: HTML5, CSS3 (Bootstrap 검토), JavaScript (필요시)
- **환경 관리**: `.venv` (가상환경), `python-dotenv`
- **주요 라이브러리**: 
  - `requests` (HTTP 통신)
  - `re` (정규표현식)
  - (향후) `Pillow` (이미지 처리), `hashlib` (해시 생성)

## 8. 보안 모범 사례 적용
- **입력값 검증**: 모든 사용자 입력에 대해 화이트리스트 기반 검증
- **XSS 방어**: Jinja2 자동 이스케이프 활용
- **CSRF 방어**: Flask-WTF 토큰 적용 검토
- **민감 정보 관리**: `.env` 파일로 API 키 관리 (Git 제외)
- **에러 핸들링**: 스택 트레이스를 사용자에게 노출하지 않음
