🚀 PRD: 0room (보안 도구 모음 웹앱)

- **정보보호 실무 지식 습득**: 암호학, HTTP 프로토콜, 웹 보안 취약점 이해

## 2. 개요 (Objective)
"0room"은 일반 사용자와 보안 입문자를 대상으로 하는, 사용자 친화적인 웹 기반 보안 유틸리티 도구 모음입니다. 복잡한 설치나 전문 지식 없이도, 누구나 일상에서 마주치는 디지털 위협을 쉽게 확인하고 예방할 수 있도록 돕는 것을 목표로 합니다.

---

## 3. 대상 고객 (Target Audience)
**일반 사용자**: 스팸 문자나 이메일로 받은 의심스러운 링크(단축 URL)를 클릭하기 전, 안전성을 확인하고 싶은 사람.

**보안/개발 입문자**: 복잡한 전문 툴(e.g., Burp Suite) 대신, 간단한 보안 기능(e.g., 암호 강도, 해시)을 빠르게 웹에서 테스트하고 싶은 학생 및 초보 개발자.

---

## 4. MVP (최소 기능 제품) 범위
"0room"의 첫 번째 버전(MVP)은 사용자가 가장 빈번하게 필요로 하는 핵심 도구 2가지를 제공합니다.

### F-01: 암호 보안 수준 검사기
**사용자 스토리**: "사용자로서, 내가 사용하려는 비밀번호를 입력하면, 이 비밀번호가 얼마나 안전한지 (e.g., 약함, 중간, 강함) 즉각적으로 피드백을 받고 싶다."

**핵심 요구사항**:
- 사용자가 암호를 입력할 수 있는 텍스트 폼(form)을 제공한다. (입력값은 type="password"로 마스킹)
- [검사하기] 버튼을 누르면 서버(Flask)로 폼 데이터가 전송(POST)된다.
- **백엔드 로직** (Python):
  - 입력된 암호의 길이를 확인한다.
  - 숫자, 대문자, 소문자, 특수문자 포함 여부를 검사한다. (정규표현식 활용)
  - 내부 기준에 따라 "약함", "중간", "강함" 등의 결과를 반환한다.
- 검사 결과를 동일 페이지 내에 텍스트로 표시한다.

**보안 고려사항**:
- 입력값 검증: XSS 공격 방지를 위한 이스케이프 처리
- 비밀번호는 서버에 저장하지 않음 (세션에도 저장 금지)

**학습 포인트**: 정규표현식, Flask form 데이터 처리, 템플릿 렌더링

---

### F-02: 안심 링크 (Safe-Click URL 확장기)
**사용자 스토리**: "사용자로서, bit.ly 같은 의심스러운 단축 URL을 입력하면, 클릭하기 전에 이 링크의 최종 목적지가 어디인지(e.g., youtube.com/...) 확인하고 싶다."

**핵심 요구사항**:
- 사용자가 URL을 입력할 수 있는 텍스트 폼을 제공한다. (type="url")
- [확인하기] 버튼을 누르면 서버(Flask)로 폼 데이터가 전송(POST)된다.
- **백엔드 로직** (Python):
  - `requests` 라이브러리를 사용해 입력된 URL에 접속(GET)한다.
  - HTTP 리다이렉션(301/302)을 자동으로 따라가 `response.url`을 통해 최종 목적지 URL을 획득한다.
  - 존재하지 않는 주소 등 예외(Exception) 상황을 처리한다.
- 최종 목적지 URL을 결과 페이지에 텍스트로 표시한다.

**보안 강화 요소 (추가 구현 예정)**:
- **HTTPS 여부 표시**: 최종 URL이 암호화된 연결인지 확인
- **도메인 유사도 검사**: 피싱 사이트 탐지 (예: `paypa1.com` vs `paypal.com`)
- **타임아웃 설정**: 악의적인 서버로 인한 무한 대기 방지

**학습 포인트**: HTTP 리다이렉션 처리, 예외 처리, requests 라이브러리 활용

## 5. 향후 확장 계획 (Roadmap / Post-MVP)
MVP 출시 이후, 다음과 같은 도구들을 순차적으로 추가합니다.

**Phase 2**:
- **해시 생성기**: 텍스트를 입력받아 MD5, SHA-256 등 주요 해시 값을 생성.
- **보안 헤더 스캐너**: 웹사이트 URL을 입력받아 HTTP 보안 헤더(e.g., X-Frame-Options, CSP) 설정 유무를 O/X로 표시.

**Phase 3**:
- **EXIF 뷰어**: 이미지 파일을 업로드하면 GPS 위치, 기기 정보 등 메타데이터를 추출해 표시.
- **(심화) "안심 링크" + VirusTotal API 연동**: F-02에서 찾은 최종 URL을 VirusTotal API로 2차 검사하여, 알려진 악성 사이트인지 여부를 함께 표시.

**Phase 4**:
- 사용자 대시보드: 검사 이력 저장 (로그인 기능 필요)
- REST API 제공: 다른 개발자가 0room 기능을 활용할 수 있도록 API 엔드포인트 공개

## 6. 기술 스택 (Tech Stack)
- **백엔드**: Python 3.x, Flask
- **프론트엔드**: HTML5, CSS3 (Bootstrap 검토), JavaScript (필요시)
- **환경 관리**: `.venv` (가상환경), `python-dotenv`
- **주요 라이브러리**: 
  - `requests` (HTTP 통신)
  - `re` (정규표현식)
  - (향후) `Pillow` (이미지 처리), `hashlib` (해시 생성)

## 8. 보안 모범 사례 적용
- **입력값 검증**: 모든 사용자 입력에 대해 화이트리스트 기반 검증
- **XSS 방어**: Jinja2 자동 이스케이프 활용
- **CSRF 방어**: Flask-WTF 토큰 적용 검토
- **민감 정보 관리**: `.env` 파일로 API 키 관리 (Git 제외)
- **에러 핸들링**: 스택 트레이스를 사용자에게 노출하지 않음
